NotEqual

概要

基本概念

非等値条件(<>)を設定します。NotEqual は、等値条件を表す ConditionKey です。

ConditionBeanについて - ConditionKeyとは？

会話上では、のっといこーる と表現します。

実装方法

実装の流れ

query() の後、set[column-name]_NotEqual() を呼び出し、条件値を指定します。 

e.g. NotEqual条件の実装手順 (Eclipseでコード補完) {MEMBER_ID} @Java
MemberCB cb = new MemberCB();
cb.q // .q と打って enter
--
cb.query()
--
// 1. .set まで打つとカラム選択
// 2. MI (MemberId) でカラム確定
// 3. NE (NotEqual) で enter
cb.query().setMINE
--
cb.query().setMemberId_NotEqual(memberId);

メソッド仕様

基本仕様

以下項目、全て Equal と同じ仕様となります。

絞り込み条件 - Equal
  • nullや空文字の指定
  • 同カラムに対する複数条件の指定
  • 同カラムに対する同じ条件値での複数指定
  • 大文字小文字の区別
  • char型でサイズ不足の値

同カラムに対して foo と bar と qux ではない、というような条件は NotInScope にて実現できます

絞り込み条件 - NotInScope

サポートされる型

  • String
  • Number 全般 (IntegerやLongなど)
  • Date 全般 (DateやTimestampなど)

Date 型に関しては、日付の非等値という条件が業務的に必要になることが少ないと考えられ、conditionBeanMap.dfprop のセットアップデフォルトでメソッドの生成が抑制されています。(コンパイルスピード向上のために)

dfprop - セットアップデフォルト
conditionBeanMap.dfprop
