おすすめのワークスペース設定

jfluteおすすめの 新しいワークスペース作ったらやっておこう設定 を紹介します。

  • DBFluteの補完テンプレートを入れる (スムーズなJava実装のために)
  • 使わないクラスをコード補完から除外する (補完ノイズを減らそう)
  • .(ドット)始まりのファイルの表示 (.settingsなどは意識したいので)
  • パッケージの見た目を階層構造へ (大量のパッケージに埋もれないように)
  • Searchダイアログのタブは File Search だけ
  • Find/Replace は Case sensitive, Incremental に
  • Tool Bar Visibility は Launch だけ
  • コンソールのバッファサイズを80万くらいに
  • fast viewうまく使ってコードの縦表示を長く
  • エディターの背景色を設定して気持ちよく

使わないクラスをコード補完から除外する

コード補完時に、例えば java.util.List を補完しようとしてるのに毎度毎度 "java.awt.Listとどっちを選ぶ？" と聞かれたりします。Type Filters にて "java.awt.*" と設定すれば、そんなことはもう聞かれません。 もう、絶対に使わないのにも関わらず、わりとよく出てくる紛らわしいものを設定しておくと良いでしょう。

Eclipseのノイズ減らし Type Filters - オススメ

.(ドット)始まりのファイルの表示

.project や .classpath, .settingsディレクトリなどは、デフォルトではPackage Explorer上に表示されません。 ですが、それらは開発者にとって重要なファイルでもありますので、常に表示すると良いでしょう。

隠しファイルの設定 - .(ドット)始まりのファイルの表示

パッケージの見た目を階層構造へ

Package Explorer上のパッケージ表現がデフォルトではフラットになっていますが、少し大きなプロジェクトになると細かいパッケージがたくさん並んで見づらいでしょう。 特にDBFluteを使っている場合は自動生成クラスがたくさんのパッケージを使っているので、階層構造に変更すると良いでしょう。

Searchダイアログのタブは File Search だけ

control+H のファイル検索 (プロジェクト全体のファイルの中身を検索) は、ほとんど File Search しか使わないのに、毎回 File Search にタブを切り替えるワンクリックが面倒だなと感じます。 そもそも最初から File Search のタブだけが表示されるようにすると良いでしょう。

Searchダイアログ - File Search だけ

Find/Replace は Case sensitive, Incremental に

command+F の Find/Replace (開いているファイルのテキストを検索) は、デフォルトでは大文字小文字を区別しません。 Javaなど大文字小文字を区別するプログラミング言語を扱っている場合、大文字小文字を意識して検索・置換したいことが多いので、Case sensitiveにチェックを付けておくと良いでしょう。 (特に、文字列置換に関しては、大文字小文字で事故も起きやすいので気を付けましょう)

Find/Replace - 入力しながら検索、Incrementalに

また、Incrementalにチェックを付けておくと、検索文字列を入力しながら検索できると打ち間違いとかもすぐにわかるので便利です。
