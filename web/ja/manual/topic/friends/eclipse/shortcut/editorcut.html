Eclipse, エディタショートカット

${indexlist}

エディタショートカットとは？

エディタとして共通的に役に立つ細かいショートカットです。

コード補完 (補完候補の表示)

ドット補完 (自然に補完)

自然に補完されるドット補完があります。ドットはメソッドを呼ぶときなどに自然と打ち込むので、そのときに補完候補が表示されます。 

e.g. ドット補完 @Java
memberBhv. // ここでドットを打った瞬間に補完候補が表示される
　↓↓↓
memberBhv.selectList() // 候補から選べば補完できる

control+space補完 (意図的に補完)

入力中でも control+space で任意のタイミングでコード補完ができます。 

e.g. control+space補完 @Java
me // ここで control+space すると補完候補が表示される
　↓↓↓
memberBhv // 候補から選べば補完できる (続けてドットを打って補完)

様々なケースで使える補完なので、Eclipseを使う場合はぜひ手に馴染ませましょう。

  • 補完候補が一つに特定できる場合は、control+spaceの瞬間に補完されます
  • クラス名の補完のときは、自動的にimport文が定義されます
  • ドット補完を途中で止めて、再度補完したいときにも使えます

2,3文字打ったらcontrol+spaceして選ぶ というスタイルがオススメです。

また、キャメルケースコード補完と相性が良いので、うまく連携させていきましょう。

プログラミング - キャメルケースコード補完

control+spaceが効かない？

特に Mac にて、control+spaceが効かないケースが報告されています。これは、Mac自体のソフトがcontrol+spaceを奪っていることが多くの原因です。

jflute個人の経験では、Spotlightがcontrol+spaceを奪っていたので、Mac購入直後はすぐにSpotlightのショートカットキーを変更します。 また、Google日本語入力がcontrol+spaceを奪っていることもあったので、キーボード環境設定でショートカットを変えています。

"mac Eclipse control+space" で検索すると詳しい記事がヒットするので参考に。

基本操作周り

行コピー
alt + command(ctrl) + 上矢印(or 下矢印)
カーソルの存在する行の内容を矢印の方向にコピーする。 複数行選択されている場合は、複数行コピーとなる。(OSレベルのショートカットとバッティングする場合も)
行移動
alt + 上矢印(or 下矢印)
カーソルの存在する行の内容を矢印の方向に移動する。 複数行選択されている場合は、複数行移動となる。
一行削除
command(ctrl) + D
どこにカーソルがあっても、そのカーソルの行を削除する。
次の行へ
shift + enter
どこにカーソルがあっても、そのカーソルの行から次の行に空行を作って移動する。
前に戻る
alt + command + 左矢印 (Windowsだとaltだけ)
前にフォーカスした箇所に戻る。(前のファイルに戻る)
command + click で別のファイルのソースコードを見た後、さっきのファイルに戻るときに使える。

ファイル周り

現在のファイルのウィンドウを最大化
control + M
現在開いているファイルのウィンドウを最大化して見やすくする。戻す時も同じ。 ファイルだけでなく、コンソールなどのウィンドウも同じように最大化できる。
現在のファイルを閉じる
command(ctrl) + W
現在開いているファイルを閉じる。
隣のファイルへ行く
control + shift + fn + 上矢印(or 下矢印)
上矢印の場合は左のファイルへ、下矢印の場合は右のファイルへ。
